# Rules for design/decisions/**

When creating or reading design decision records:

1. **Each DDR must reference specific HCI principles** — not just "looks good"
2. **Include severity assessment** — using Nielsen's 0-4 scale where applicable
3. **Document trade-offs** — what was gained and what was sacrificed
4. **Name the deciding factor** — which principle or user need tipped the decision
5. **Existing DDRs are the project's design history** — read before proposing alternatives
6. **FINAL-DESIGN-SYNTHESIS.md is the canonical direction** — proposals feed into this
